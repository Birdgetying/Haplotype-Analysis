## 发现的问题

### 问题1: 数据库annotation字段完全错误 ❌ → ✅ 已修复
**现象**:
- 所有CDS内的SNP都被标注为 `intron`（应该是 missense/synonymous/other）
- 启动子区域的SNP被标注为 `other`（应该是 `promoter`）
- 完全没有 missense/synonymous 分类

**根本原因**:
数据库构建时（`genome_wide_haplotype_scan.py` 行1031），代码尝试访问全局变量 `FASTA_FILE`，但该变量定义在 `ScanConfig` 类中，不在全局作用域。导致：
```python
_fasta_path = None  # 因为 NameError 被捕获
try:
    _fasta_path = FASTA_FILE  # ← NameError!
except NameError:
    pass
```

结果 `annotate_snp_effects_for_region` 收到 `fasta_path=None`，走入fallback逻辑，但fallback逻辑中 `in_exon` 判断也可能失败，导致CDS内SNP被错误标注为 `intron`。

**修复**:
运行 `reannotate_database.py` 重新注释了所有31个基因的 `variant_info.csv`

### 问题2: FASTA路径传递错误 ⚠️ 待修复
**现象**:
`genome_wide_haplotype_scan.py` 中使用 `FASTA_FILE`（全局变量），但该变量定义在 `ScanConfig.FASTA_FILE`

**影响**:
数据库重建时无法获取FASTA，无法进行 missense/synonymous 注释

**修复方案**:
```python
# 修改前（行1031）
_fasta_path = FASTA_FILE

# 修改后
_fasta_path = getattr(ScanConfig, 'FASTA_FILE', None)
```
